# Security

MitLicense Center — внутренняя админ-панель для одного доверенного оператора на
**одноузловом** Windows-сервере (backend + IIS + MSSQL + 1С-кластер на одном
хосте). Модель безопасности исходит из этого: аутентификация людей делегирована
периметру сети (LAN/VPN, AD/SSO на краю), а сама панель защищает данные и
операции, а не сетевую границу. Подробности — в `docs/DECISIONS.md` и
`docs/OPERATIONS.md`.

## Ключевые решения
- **Секреты в покое — DPAPI** (ASP.NET Data Protection), не plaintext и не Vault — ADR-8. Key ring + БД образуют единый бэкап-юнит (`docs/OPERATIONS.md`).
- **Принятый риск: пароль кластера 1С передаётся в командной строке `rac.exe`** — поддерживаемые версии не дают иного канала. Компенсируется одноузловой admin-only средой и DPAPI в покое. ADR-21.
- **Hardening включается конфигом, не хардкодом:** HTTPS-redirect/HSTS, шифрование SQL-канала, Swagger, AllowedHosts — через ключи `Security:*` и `appsettings.Production.json`. ADR-22 и `docs/OPERATIONS.md` §«Transport hardening».
- **Роли** `Admin` / `Viewer` проверяются на сервере; пароли — политика Identity (12+ символов, lockout 5 попыток / 15 мин).
- **Намеренно вне зоны (ADR-15):** встроенная 2FA и оркестрация бэкапов — за периметром сети / на операторе.

## Сообщить об уязвимости
Проект внутренний. Сообщать сопровождающему репозитория напрямую (приватно), не
публичным issue. Не коммить секреты, дампы БД и DPAPI key ring.
