## E. Как достать тело статьи ИТС (curl-рецепт)

Статьи ИТС читаются **без авторизации**, если брать прямой адрес тела статьи
(`.../src/.../i8NNNNNN.htm`). Хэш-ссылки `/db/.../content/NNNN/hdoc` через WebFetch
дают только меню и кракозябры — нужен curl + перекодировка из `windows-1251`.

Тело статьи — числовые HTML-сущности (`&#NNNN;`), декодируются так:

```python
# fetch1c.py — печатает читаемый текст статьи ИТС
import sys, urllib.request, html, re
url = sys.argv[1]
req = urllib.request.Request(url, headers={'User-Agent': 'Mozilla/5.0'})
raw = urllib.request.urlopen(req, timeout=30).read()
txt = raw.decode('windows-1251', 'replace')
txt = re.sub(r'(?is)<(style|script).*?</\1>', ' ', txt)
txt = re.sub(r'(?is)<[^>]+>', ' ', txt)
txt = re.sub(r'[ \t]+', ' ', html.unescape(txt))
print(re.sub(r'\n\s*\n+', '\n', txt).strip())
```

Запуск (Windows, важно UTF-8 stdout):
```bash
PYTHONIOENCODING=utf-8 python fetch1c.py "https://its.1c.ru/db/content/metod8dev/src/developers/scalability/methods/i8105927.htm"
```

Соответствие `hdoc → .htm` не 1:1 по номеру. Если на руках только `content/NNNN/hdoc`,
а нужен `.htm`: открыть hdoc, найти в нём iframe/ссылку на `.../src/.../i8NNNNNN.htm`,
либо искать в вебе по заголовку статьи (см. имеющиеся .htm выше как образец путей).
