# MitLicense Center — обзор продукта

MitLicense Center — веб-панель управления мультитенантной 1С-инфраструктурой
для хостинг-провайдера, работающего на Windows Server + IIS + SQL Server.

---

## 1. Какую проблему решает продукт

Хостинг-провайдер держит один или несколько серверов 1С:Предприятие,
на которых живут информационные базы десятков клиентов. Каждому клиенту выделена
квота одновременных пользовательских лицензий. Без централизованного инструмента
оператор не видит: кто сколько лицензий потребляет прямо сейчас, превышает ли
кто-то свой лимит, опубликована ли база в IIS и на какой версии платформы.

MitLicense Center решает три задачи:

1. **Учёт и контроль лицензий.** Панель непрерывно опрашивает кластер 1С через
   `rac.exe`, считает активные сеансы по каждому клиенту и автоматически завершает
   лишние сеансы (enforcement), как только лимит превышен — без участия оператора.
   История потребления хранится с шагом 15 минут и доступна в разделе «Отчёты».

2. **Управление публикациями IIS.** Оператор публикует инфобазу в IIS прямо
   из панели через `webinst.exe`, меняет версию платформы в `web.config`,
   проверяет текущее состояние публикации и управляет пулами приложений и сайтами
   (recycle, stop, iisreset) — не заходя на сервер вручную.

3. **Операционная прозрачность.** Раздел «Быстродействие» показывает загрузку
   процессора, памяти и дисков хоста «сейчас», раскладывает нагрузку по семьям
   процессов (1С, SQL Server, антивирус, прочее), показывает активные сеансы
   кластера и активные запросы SQL. Аудитный журнал фиксирует все изменения.
   On-demand бэкап базы SQL запускается в один клик.

## 2. Целевые пользователи

**Оператор хостинга / системный администратор** — основной пользователь.
Работает с ролью **Admin**: полный доступ ко всем операциям.
Управляет клиентами, инфобазами, публикациями IIS, параметрами системы,
учётными записями панели; завершает сеансы вручную; запускает бэкапы и удаляет их;
управляет пулами и сайтами IIS.

**Наблюдатель / дежурный** — пользователь с ролью **Viewer**.
Видит все данные (дашборд, сеансы, инфобазы, отчёты, аудит, бэкапы, быстродействие),
но не может изменять конфигурацию и выполнять разрушительные действия.
Исключение: запуск on-demand бэкапа доступен обеим ролям (Admin и Viewer).

Роль назначается при создании учётной записи и может быть изменена администратором.
Страницы «Параметры» и «Пользователи» открываются только пользователям с ролью Admin
(маршрутная защита на фронтенде + политика авторизации на API).

## 3. Границы продукта

MitLicense Center **не** является:

- **Биллинговой системой** — не выставляет счета, не считает деньги, не интегрируется
  с платёжными шлюзами.
- **Десктоп-приложением** — работает только через браузер; сервер — служба Windows.
- **Многоузловым кластерным менеджером** — панель управляет тем узлом,
  на котором установлена (single-host). Один SQL-инстанс задаётся в параметрах;
  один RAS-эндпоинт подключается к локальному `ras.exe`.
- **Средством резервного копирования «под ключ»** — бэкапы on-demand и COPY_ONLY,
  без планировщика и ротации цепочек дифференциальных резервных копий.
- **Инструментом конфигурирования 1С** — не создаёт и не удаёт базы в кластере,
  не управляет пользователями 1С, не трогает конфигурации.

## 4. Состав возможностей

Панель организована в навигационные группы:

### Мониторинг

| Страница | Путь | Что показывает |
|---|---|---|
| Обзор | `/` | KPI-карточки: клиенты, инфобазы, активные сеансы, использование и остаток лицензий, статус RAS; топ клиентов по нагрузке; метрики хоста |
| Сеансы | `/sessions` | Активные сеансы 1С в реальном времени (обновление каждые 5 с); ручное завершение сеанса; фильтр по клиенту/инфобазе |
| Отчёты | `/reports` | График потребления лицензий во времени с шагом 15 мин; сводка по всем клиентам и детализация по одному; экспорт CSV/Excel/HTML/PDF |
| Быстродействие | `/performance` | Загрузка CPU/RAM/диска хоста; атрибуция нагрузки по семьям процессов; активные сеансы и рабочие процессы кластера 1С; активные запросы SQL, ожидания, задержки диска; запись по требованию |

### Управление

| Страница | Путь | Что делает |
|---|---|---|
| Клиенты | `/tenants` | CRUD клиентов; лимиты лицензий; список инфобаз клиента |
| Базы | `/infobases` | CRUD инфобаз; публикация/снятие через webinst; смена версии платформы; проверка статуса IIS; бэкапы базы; разбор нераспределённых баз кластера; управление пулами и сайтами IIS (вкладка IIS) |

### Система

| Страница | Путь | Что делает |
|---|---|---|
| Аудит | `/audit` | Хронологический журнал всех операций с фильтрами по типу действия, клиенту и дате |
| Пользователи | `/users` | CRUD учётных записей панели; сброс пароля; смена роли Admin↔Viewer; отключение/включение (только Admin) |
| Параметры | `/settings` | Подключение к 1С/RAS, SQL Server, IIS, политика лицензий, частота опроса, хранение данных, бэкапы (только Admin) |

## 5. Глоссарий

Эталонная терминология для документации v2 и UI-текстов.
Источники: `frontend/src/i18n/ru.json`, доменный и прикладной код backend
(`MitLicenseCenter.Domain`, `MitLicenseCenter.Application`, контракты эндпоинтов).

| Термин (рус.) | Термин в коде / API | Определение |
|---|---|---|
| **Клиент** / тенант | `Tenant`, поле `name`; API `/api/v1/tenants` | Арендатор 1С-хостинга. Имеет название, лимит лицензий и статус активности. Одному клиенту принадлежат одна или несколько инфобаз. |
| **Инфобаза** | `Infobase`; API `/api/v1/infobases` | Информационная база 1С:Предприятие. Хранится в кластере 1С (идентифицируется `ClusterInfobaseId` — GUID) и в SQL Server (поле `DatabaseName`). Принадлежит ровно одному клиенту. |
| **Публикация** | `Publication`; поля `SiteName`, `VirtualPath`, `PlatformVersion`; API `/api/v1/publications` | Запись о том, как инфобаза опубликована на IIS: сайт, виртуальный путь, версия платформы 1С. Одна инфобаза — одна публикация. |
| **Статус публикации** | `PublicationPublishStatus` (`Unknown`, `Published`, `NotPublished`, `Error`) | Результат последней проверки факта публикации в IIS (наличие сайта, виртуального каталога, web.config). Обновляется вручную («Проверить») или фоново каждые 5 минут. |
| **Источник публикации** | `PublicationSource` (`Unknown`, `Webinst`, `Configurator`) | Происхождение публикации: `Webinst` — создана панелью через `webinst.exe`; `Configurator` — создана вне панели (вручную или через Конфигуратор). Влияет на поведение при повторной публикации: перезапись `Configurator`-публикации требует явного подтверждения. |
| **Сеанс** | `ClusterSession`; API `GET /api/v1/sessions/snapshot` | Активное подключение пользователя (или фонового процесса) к инфобазе в кластере 1С. Сеанс потребляет лицензию, если его `app-id` входит в настройку `OneC.LicenseConsumingAppIds`. |
| **Снимок сеансов** | `SnapshotPayload`; `GET /api/v1/sessions/snapshot` | Актуальный срез всех активных сеансов кластера, полученный от `rac.exe`. Используется для отображения в разделе «Сеансы» и как базис расчёта потребления при enforcement. |
| **Лимит лицензий** | `Tenant.MaxConcurrentLicenses`; `audit.actions.LimitChanged` | Максимальное число одновременных лицензионных сеансов для клиента. `0` означает отсутствие лимита (сеансы считаются, но enforcement не применяется). |
| **Контроль квот** / enforcement | `IKillEnforcer`, `IEnforcementGate`; `AuditReason.LimitExceeded` | Автоматическое завершение избыточных сеансов при превышении лимита лицензий. Сеансы завершаются по принципу «newest-first» до восстановления нормы. Отсрочка перед завершением задаётся параметром `Enforcement.KillGraceSeconds`. |
| **Кластер 1С** | `OneC.RAS.Endpoint`; `IClusterClient` | Сервер 1С:Предприятие, к которому подключается панель. Взаимодействие — через `rac.exe` (утилита командной строки, ADR-3.3). |
| **RAS** | `OneC.RAS.Endpoint` (формат `host:port`); `OneC.RAS.ExePath` | Remote Administration Service — служба 1С, предоставляющая API управления кластером. Панель подключается к локальному `ras.exe` через `rac.exe`. |
| **rac.exe** | `OneC.RAS.ExePath` | Утилита командной строки 1С для управления кластером через RAS. Путь задаётся в параметрах. Вывод декодируется в CP866 (OEM-кодировка на RU Windows). |
| **Discovery** | API `/api/v1/discovery/*`; компонент `DiscoveryField` | Автоматическое обнаружение объектов: базы кластера 1С, базы SQL Server, сайты IIS, пути к `rac.exe`, версии платформы, SQL-инстансы. Используется для заполнения выпадающих списков в формах. |
| **Нераспределённая база** | `HiddenClusterInfobase`; API `/api/v1/infobases/unassigned`; i18n `infobases.unassigned` | База, существующая в кластере 1С, но не заведённая в панель. Её сеансы не учитываются в лимитах. Оператор может назначить её клиенту («Назначить») или скрыть из баннера («Скрыть»). |
| **Скрытая база** | `HiddenClusterInfobase`; действия `hide`/`unhide` | Нераспределённая база кластера, намеренно исключённая из баннера-счётчика. Запись хранится в БД со снапшотом имени, чтобы список отображался даже при недоступном RAS. |
| **Снимок потребления** | `LicenseUsageSnapshot` (таблица `LicenseUsageSnapshots`); API `/api/v1/reports/license-usage` | Агрегат потребления лицензий клиента за закрытый 15-минутный бакет (min/max/avg). Фоновое задание `cold-snapshot` семплирует потребление каждый цикл, аккумулятор копит running-агрегаты и пишет строку на границе бакета — раз в 15 минут. Питает раздел «Отчёты». |
| **Бэкап** | `Backup`; API `/api/v1/backups`; `BackupStatus` (`Queued`, `Running`, `Succeeded`, `Failed`) | On-demand полная резервная копия базы SQL (`BACKUP DATABASE … COPY_ONLY`). Запускается с карточки инфобазы. Хранится один актуальный файл на базу; устаревшие удаляются автоматически ночью. |
| **Аудит** | `AuditActionType`; API `/api/v1/audit` | Неизменяемый хронологический журнал всех значимых операций системы: создание/изменение/удаление объектов, входы, смены ролей, kill сеансов, операции IIS, бэкапы, изменения параметров. |
| **Роль Admin** | `RequireRole("Admin")`; i18n `users.roles.Admin` | Полный доступ: управление объектами, разрушительные операции, параметры, пользователи. |
| **Роль Viewer** | `RequireRole("Admin","Viewer")`; i18n `users.roles.Viewer` | Только просмотр: все разделы видны, но кнопки создания/изменения/удаления скрыты (кроме запуска бэкапа, доступного обеим ролям). |
| **Параметры** | `SettingKey`; API `/api/v1/settings`; страница «Параметры» | Runtime-настройки системы: подключение к RAS, SQL, IIS, политика лицензий, частота опроса, сроки хранения, параметры бэкапа. Хранятся в БД; значения секретных настроек шифруются через ASP.NET Data Protection (protector, purpose `mlc.settings.v1`), key ring лежит в файловой системе без шифрования (защита — NTFS ACL). |
| **Версия платформы** | `Publication.PlatformVersion`; `OneC.DefaultPlatformVersion` | Версия 1С:Предприятие в формате `Мажор.Минор.Билд.Ревизия` (например `8.3.23.1865`). Определяет путь к `wsisapi.dll` в `web.config` публикации IIS. |
| **Пул приложений IIS** | `IisAppPoolInfo`; API `/api/v1/iis/application-pools`; i18n `publications.iis.pools` | Application Pool IIS, в котором работает опубликованная инфобаза. Панель поддерживает recycle, start, stop. |
| **Сайт IIS** | `IisSiteInfo`; `Publication.SiteName`; API `/api/v1/iis/sites` | Сайт IIS, на котором размещается виртуальный каталог инфобазы. Задаётся по умолчанию в параметре `IIS.DefaultSiteName`; для конкретной инфобазы переопределяется в публикации. |
| **Виртуальный путь** | `Publication.VirtualPath` | Путь приложения IIS вида `/имя-базы` (например `/acme-bp`). Вместе с именем сайта образует URL публикации. |
| **webinst** | `PublicationSource.Webinst`; утилита `webinst.exe` | Штатная утилита 1С для публикации информационной базы на веб-сервере. Создаёт `default.vrd` и `web.config`. Панель вызывает `webinst.exe` при публикации и снятии публикации. |
| **default.vrd** | файл на диске IIS-хоста | Файл описания публикации 1С (Virtual Resource Description). Создаётся `webinst.exe` при публикации. |
| **Статус базы** | `InfobaseStatus` (`Active`, `Maintenance`, `Suspended`) | Операционный статус инфобазы в панели: `Active` — штатная работа, `Maintenance` — обслуживание, `Suspended` — приостановлена. Не связан с состоянием в кластере 1С. |
| **Circuit breaker** | `AuditActionType.ClusterAdapterCircuitOpened/Closed` (исторические записи) | Ранее применявшийся механизм защиты от каскадных сбоев при недоступности кластера 1С. Удалён; enum-значения сохранены для отображения исторических записей аудита. |
| **Фоновое задание** | Hangfire recurring job | Периодически выполняемая задача бэкенда: `cold-snapshot` (каждую минуту) — опрос сеансов и enforcement; `publication-status-refresh` (каждые 5 мин) — обновление статуса публикаций IIS; ночные задания — очистка аудита, истории лицензий, файлов бэкапов. |
| **App-id** | `OneC.LicenseConsumingAppIds` | Идентификатор типа клиентского приложения 1С (например `1CV8`, `WebClient`, `Designer`). Определяет, считается ли сеанс потребляющим лицензию. |
