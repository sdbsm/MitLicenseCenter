## 1. Вход и роли

### Страница входа

Перейдите по адресу панели в браузере. На экране «Вход в систему» введите **Имя пользователя** и **Пароль**, затем нажмите **Войти**.

При неверных данных появится сообщение «Неверное имя пользователя или пароль.». Сброс пароля выполняет администратор через раздел «Пользователи».

### Принудительная смена пароля при первом входе

Если учётная запись создана администратором или пароль был сброшен, система показывает блокирующий экран «Смените пароль». На него невозможно обойти навигацией — пока пароль не сменён, остальные разделы недоступны.

Заполните три поля: **Текущий пароль** (временный, выданный администратором), **Новый пароль**, **Повторите новый пароль**. Требования к новому паролю: не короче 12 символов, содержит заглавную и строчную буквы, цифру и спецсимвол. Нажмите **Сменить и продолжить** — блокирующий экран исчезнет.

### Смена своего пароля в любой момент

В верхней части панели нажмите на имя своей учётной записи → выберите **Смена пароля**. В открывшемся диалоге заполните поля **Текущий пароль**, **Новый пароль**, **Повторите новый пароль** и нажмите **Сменить пароль**.

### Роли и доступ

| Роль | Что доступно |
|---|---|
| **Администратор** | Все разделы и операции: создание/изменение/удаление клиентов, инфобаз и публикаций; управление пользователями и параметрами; завершение сеансов вручную; операции IIS; удаление бэкапов. |
| **Наблюдатель** | Все разделы доступны для просмотра; кнопки создания, изменения, удаления скрыты. Может запускать бэкап (кнопка «Сделать бэкап» доступна обеим ролям). |

Разделы **Параметры** (`/settings`) и **Пользователи** (`/users`) открываются только администратору. Если Наблюдатель попытается перейти по URL — система перенаправит на главную страницу.

## 2. Сквозной сценарий онбординга клиента

Полный путь: создать клиента → создать инфобазу → опубликовать в IIS → проверить статус публикации.

### Шаг 1. Создать клиента

1. Перейдите в раздел **Клиенты** (левое меню → группа «Управление»).
2. Нажмите **Добавить клиента**.
3. Заполните форму:
   - **Название** — уникальное имя арендатора, например «ООО Ромашка». Не длиннее 200 символов.
   - **Лимит лицензий** — максимальное число одновременных лицензионных сеансов на все базы этого клиента. Введите `0`, если лимит не нужен (сеансы считаются, но принудительного завершения не будет).
   - Флаг **Активен** — снимите, если клиент временно не должен учитываться в лимитах.
4. Нажмите **Создать**. Клиент появится в списке. Сообщение «Клиент «...» создан.» подтверждает успех.

### Шаг 2. Создать инфобазу и привязать к клиенту

1. Перейдите в раздел **Базы** или откройте карточку клиента (кликните по строке в разделе «Клиенты») и нажмите **Добавить инфобазу**.
2. В форме **Новая инфобаза** заполните три основных поля:
   - **Клиент** — выберите из выпадающего списка клиента, созданного на шаге 1. После создания инфобазы клиента сменить нельзя.
   - **База в кластере 1С** — выберите базу из автоматически обнаруженного (discovery) списка баз кластера. Система подставит название и GUID базы (`ID в кластере 1С`) автоматически. Если discovery недоступен, переключитесь на «Ввести вручную» и введите GUID.
   - **Имя базы данных (SQL Server)** — выберите из discovery-списка баз SQL-инстанса или введите вручную (например `acme_bp`).
3. При необходимости раскройте блок **Дополнительно** — он содержит две группы. Группа **Инфобаза**:
   - **Название** — отображаемое имя в панели. По умолчанию подставляется из выбранной базы кластера — можно изменить.
   - **Статус** — операционный статус базы (`Активна`, `Обслуживание`, `Приостановлена`).

   Группа **Публикация в IIS**:
   - **Сайт IIS** — по умолчанию берётся из параметра `IIS.DefaultSiteName`.
   - **Виртуальный путь** — путь вида `/имя-базы` (например `/acme-bp`), начинается с `/`, без пробелов.
   - **Версия платформы 1С** — четыре числовых сегмента, например `8.3.23.1865`. По умолчанию из параметра `OneC.DefaultPlatformVersion`.
   - **Физический путь** — необязательное переопределение каталога публикации; по умолчанию вычисляется из имени базы.
4. Нажмите **Создать**. Инфобаза зарегистрирована в панели.

### Шаг 3. Опубликовать инфобазу в IIS

1. В разделе **Базы** найдите созданную инфобазу и нажмите кнопку **Опубликовать** (или раскройте строку → «Опубликовать»).
2. Откроется диалог **Опубликовать через webinst**. Для подтверждения введите токен — строку `Сайт+виртуальный_путь` (например `Default Web Site/acme-bp`), указанную в подсказке диалога, и нажмите **Опубликовать**.
   > Если публикация по этому пути уже существует и была создана вне панели (источник «Конфигуратор»), диалог дополнительно предупредит, что перезапись через webinst удалит ручные настройки (`default.vrd`, `web.config`).
3. Панель вызывает `webinst.exe`, создаёт `default.vrd` и `web.config`. При успехе появляется уведомление «Инфобаза опубликована».

### Шаг 4. Проверить статус публикации

- Столбец **Публикация** в списке баз показывает один из статусов: **Опубликована**, **Не опубликована**, **Ошибка проверки**, **—** (не проверялась).
- Нажмите **Проверить публикацию** на строке базы, чтобы запросить немедленную проверку. Фоновое обновление происходит автоматически каждые 5 минут.
- Если статус — «Ошибка проверки», проверьте доступность IIS и права службы (раздел «Параметры» → секция «Публикации IIS»).

## 3. Лимиты лицензий

### Как задаётся лимит

Лимит задаётся в карточке клиента, поле **Лимит лицензий** (`MaxConcurrentLicenses`). Значение `0` означает отсутствие лимита: сеансы продолжают учитываться в статистике, но принудительное завершение (enforcement) не срабатывает.

Изменить лимит: Клиенты → выбрать клиента → **Изменить** → поле «Лимит лицензий» → **Сохранить**. Событие фиксируется в журнале аудита как «Лимит изменён».

### Что считается лицензионным сеансом

Сеанс потребляет лицензию, если кластер 1С реально выдал ему **клиентскую лицензию**. Панель определяет это по **факту от 1С** (команда `rac --licenses`), а не по типу приложения. Фоновые задания клиентскую лицензию не держат и в лимит не входят.

В таблице раздела «Сеансы» столбец **Лицензия** показывает **«Считается»**, **«Не считается»** или **«Определяется…»** (для только что подключившегося сеанса, пока факт ещё не получен — обычно несколько секунд).

> Если связь с 1С временно недоступна и факт о лицензиях получить не удаётся, панель показывает «Данные о лицензиях недоступны» и на это время **не завершает** сеансы по лимиту — чтобы не отключить пользователя вслепую.

### Где видно текущее потребление

- **Дашборд (Обзор)**: карточки «Использовано лицензий» и «Свободно лицензий» — суммарные значения по всем клиентам; блок «Топ клиентов по нагрузке» показывает прогресс-бар `потреблено / лимит` и процент заполнения для каждого клиента с ненулевым лимитом.
- **Раздел «Сеансы»**: реальное число активных сеансов по инфобазам с фильтром по клиенту; данные обновляются каждые 5 секунд.
- **Раздел «Отчёты»**: история потребления с шагом 15 минут; сводка по всем клиентам или детализация по одному; кривые пика и среднего; строка «Пик за период / Среднее за период».

### Как читать прогресс и превышение

Прогресс-бар в «Топ клиентов»:
- зелёный — до 75% заполнения;
- жёлтый (amber) — 75–89%;
- красный — 90% и выше.

В «Отчётах» для выбранного клиента на графике отображается линия **Лимит** — горизонтальная пунктирная черта. Бары выше этой линии означают, что в данном 15-минутном интервале потребление превышало лимит.

## 4. Нарушение лимита и реакция системы

### Автоматический контроль квот (enforcement)

Когда число лицензионных сеансов клиента превышает его лимит, панель автоматически завершает лишние сеансы — без участия оператора.

Механизм:
1. Фоновый cold-цикл (`ColdTierPollingService`) опрашивает кластер 1С каждые `Polling.ColdIntervalSeconds` секунд (по умолчанию 15 с). Когда заполнение достигает порога `Polling.HotThresholdPercent` (по умолчанию 90%), база переходит на ускоренный режим опроса каждые `Polling.HotIntervalSeconds` секунд (по умолчанию 4 с).
2. При обнаружении превышения система выдерживает паузу `Enforcement.KillGraceSeconds` (по умолчанию 15 с), чтобы 1С успел зарегистрировать имя пользователя.
3. Лишние сеансы завершаются в порядке «сначала новые» (newest-first), пока потребление не вернётся в норму.

### Где это видно

- **Раздел «Сеансы»**: сеанс, завершённый enforcement-ом, исчезает из таблицы после следующего обновления (15 с).
- **Раздел «Аудит»**: каждое принудительное завершение фиксируется записью «Сеанс завершён» с причиной превышения лимита (`AuditReason.LimitExceeded`), именем пользователя и именем инфобазы.

### Ручное завершение сеанса оператором

Только **Администратор** может завершить сеанс вручную.

1. Перейдите в **Сеансы**.
2. В строке нужного сеанса нажмите кнопку завершения (иконка в последнем столбце).
3. В диалоге «Завершить сеанс?» введите в поле подтверждения идентификатор приложения (`appId`), указанный в подсказке.
4. Опционально заполните поле **Причина** (например «плановое обслуживание»).
5. Нажмите **Завершить**. Если сеанс уже завершился к этому моменту — появится сообщение «Сеанс уже завершён.».

Ручное завершение также фиксируется в разделе «Аудит» как «Сеанс завершён».

## 5. Разделы панели

### 5.1. Обзор (дашборд)

Маршрут `/` (главная страница).

**Что здесь:** сводная картина состояния системы «прямо сейчас».

- Пять KPI-карточек: **Клиенты** (активных из всех), **Инфобазы** (всего), **Активные сеансы**, **Использовано лицензий**, **Свободно лицензий**. Каждая карточка кликабельна и ведёт в профильный раздел.
- Карточка **Статус RAS**: показывает «OK», «Сбой» или «Проверка…»; при сбое — ссылка «детали ошибки» с текстом последней ошибки в тултипе.
- Карточка **Здоровье хоста**: процессор, память, диск в виде компактных индикаторов.
- Блок **Топ клиентов по нагрузке**: список клиентов с прогресс-баром потребления и значением `потреблено / лимит (%)`. Клик по имени ведёт на карточку клиента.

**Действия оператора:** мониторинг общего состояния; переход в нужный раздел по карточкам.

---

### 5.2. Клиенты

Маршрут `/tenants`, карточка клиента — `/tenants/:id`.

**Что здесь:** список всех клиентов хостинга с лимитами лицензий и числом инфобаз.

**Доступные действия (Admin):**
- **Добавить клиента** — форма с полями «Название», «Лимит лицензий», флаг «Активен».
- **Изменить** — редактирование названия, лимита и статуса активности.
- **Удалить** — только если у клиента нет инфобаз (сначала удалите базы). Требует ввода названия клиента для подтверждения.

**Карточка клиента** (`/tenants/:id`): список инфобаз клиента, лимит, кнопки «Добавить инфобазу» и «Изменить».

**Viewer** видит список и карточки, кнопки изменения скрыты.

---

### 5.3. Базы

Маршрут `/infobases`, вкладки **Базы** и **IIS**.

**Что здесь:** все зарегистрированные инфобазы с фильтром по клиенту и статусу публикации.

**Вкладка «Базы» — доступные действия (Admin):**
- **Добавить инфобазу** — полная форма (см. раздел 2, шаг 2).
- **Изменить** — поля названия, статуса базы (`Активна`, `Обслуживание`, `Приостановлена`), параметры публикации в блоке «Дополнительно».
- **Перенести в другого клиента** — dialog «Перенести инфобазу», выбор нового клиента; публикация IIS при переносе не меняется.
- **Удалить** — с опцией «Также снять публикацию из IIS». Требует ввода названия инфобазы.
- **Опубликовать / Снять публикацию / Проверить публикацию / Сменить платформу** — операции с публикацией IIS для конкретной базы. Смена версии платформы перезаписывает только путь к `wsisapi.dll` в `web.config`; файл `default.vrd` и пользовательские настройки публикации не затрагиваются, полная повторная публикация не требуется.
- **Бэкапы базы** — открывает диалог бэкапов (см. раздел 6).

**Массовые операции:** выберите несколько строк (чекбоксы) → появится панель **Выбрано публикаций: N** с кнопками **Опубликовать выбранные** и **Сменить платформу выбранным**. Прогресс отображается в реальном времени.

**Баннер нераспределённых баз:** если в кластере 1С есть базы, не заведённые в панель, вверху страницы появляется предупреждение «N баз кластера не заведено в панель — их сеансы не считаются в лимиты». Нажмите **Разобрать** — откроется диалог со списком нераспределённых баз. Для каждой базы можно:
- **Назначить** — привязать к клиенту (открывает форму «Новая инфобаза», GUID подставлен).
- **Скрыть** — убрать из баннера (база остаётся нераспределённой, но счётчик её не учитывает). Ранее скрытую базу можно **Вернуть**.

**Вкладка «IIS»** — управление пулами приложений и сайтами (см. раздел 7).

### 5.4. Сеансы

Маршрут `/sessions`.

**Что здесь:** таблица всех активных сеансов 1С в реальном времени. Данные обновляются каждые 5 секунд.

Столбцы: **Клиент**, **Инфобаза**, **Идентификатор** (session-id), **Приложение** (`app-id`), **Пользователь**, **Начало**, **Длительность**, **Лицензия** (считается / не считается / определяется).

**Фильтры:** поиск по клиенту или пользователю, фильтр по инфобазе.

**Действия (Admin):** кнопка завершения сеанса в каждой строке. Подробнее — раздел 4.

---

### 5.5. Отчёты

Маршрут `/reports`.

**Что здесь:** история потребления лицензий с шагом 15 минут.

**Секции страницы:**
- **Сводка по всем клиентам** — таблица с пиком потребления каждого клиента за выбранный период; кнопка навигации по месяцам.
- **Детализация по клиенту** — выберите клиента из выпадающего списка; появится линейный график с кривыми «Пик потребления», «Среднее потребление» и горизонтальной линией «Лимит»; блок статистики «Пик за период» и «Среднее за период».
- **Фильтр периода** — поля «С даты» и «По дату». Максимальный запрашиваемый период ограничен сроком хранения (параметр `LicenseUsage.RetentionDays`, по умолчанию 365 дней); если выбранный диапазон шире, отображаются последние N дней.

**Экспорт** (кнопка **Скачать**): **CSV**, **Excel**, **HTML (интерактивный)**, **PDF** — применяется к текущей детализации по выбранному клиенту и периоду.

### 5.6. Быстродействие

Маршрут `/performance`.

**Что здесь:** состояние хоста «прямо сейчас»; данные не сохраняются (кроме записей). Обновление каждые 5 секунд, пока вкладка открыта.

**Секции страницы:**
- **Светофор ресурсов** — три индикатора: Процессор, Память, Диск (латентность). Зелёный = в норме, жёлтый = нагрузка, красный = насыщение.
- **Вердикт** — текстовый итог: «Ресурсы в норме» или «Высокая нагрузка: [ресурс]. Основной потребитель — [семья процессов]».
- **Кто потребляет** — таблица семей процессов (1С, SQL Server, Антивирус, Обновления ОС, Прочее) с долями в CPU и памяти.
- **Кто грузит внутри 1С** — активные сеансы и рабочие процессы кластера 1С: пользователь, хост, приложение, CPU, длительность, состояние (Заблокирован / Долгий вызов / Активен / Молчит / Простаивает).
- **Нагрузка на SQL Server** — активные запросы MSSQL, нагрузка по базам, конкуренция за ресурсы (ожидания, задержки диска). Требует права `VIEW SERVER STATE` для учётной записи панели.
- **Запись по требованию** — кнопка **Начать запись** (только Admin): бэкенд собирает сэмплы host-метрик и «виновников» в таблицу, пока запись не остановлена или не сработает авто-стоп. Записи переживают закрытие вкладки. Для каждой записи доступны просмотр графика и экспорт (CSV, Excel); удаление — только Admin.

**Действия с записями:** запуск, остановка и удаление — только **Администратор**. **Наблюдатель** видит список записей, открывает график и экспортирует данные, но не может запускать, останавливать или удалять записи. Индикатор «Идёт запись» виден обеим ролям.

---

### 5.7. Аудит

Маршрут `/audit`.

**Что здесь:** неизменяемый хронологический журнал всех значимых событий системы. Записи нельзя редактировать или удалять вручную; очистка по сроку хранения происходит автоматически ночью.

Столбцы: **Время**, **Действие**, **Инициатор**, **Клиент**, **Описание**.

**Фильтры:** тип действия (все типы из выпадающего списка), клиент, период (С даты / По дату), размер страницы.

Типы событий включают: создание/изменение/удаление клиентов, инфобаз и публикаций; операции с пользователями (создание, смена роли, отключение, сброс пароля); вход/выход; завершение сеансов; операции IIS; бэкапы; изменения параметров; очистки журнала и истории.

Если выбранный диапазон выходит за срок хранения — система показывает предупреждение «Записи аудита старше [дата] удаляются автоматически».

---

### 5.8. Пользователи (только Admin)

Маршрут `/users`.

**Что здесь:** управление учётными записями панели.

**Действия:**
- **Добавить пользователя** — укажите логин (уникальный) и выберите роль (Администратор / Наблюдатель). Система генерирует временный пароль автоматически и показывает его **один раз** — скопируйте и передайте пользователю. При первом входе система потребует сменить пароль.
- **Сбросить пароль** — генерируется новый временный пароль; старый перестаёт действовать. Диалог показывает новый пароль один раз.
- **Сменить роль** — выберите новую роль в диалоге. Нельзя менять роль своей учётной записи и разжаловать последнего активного администратора. Новая роль вступает в силу при следующем входе пользователя.
- **Отключить / Включить** — отключённая учётная запись не может входить в систему. Нельзя отключить себя и последнего активного администратора.

---

### 5.9. Параметры (только Admin)

Маршрут `/settings`.

**Что здесь:** runtime-настройки системы. Изменения применяются без перезапуска сервиса (кроме пути к `rac.exe`, вступающего в силу при следующем опросе).

**Секции:**

| Секция | Ключевые параметры |
|---|---|
| Подключение к 1С / RAS | Логин и пароль администратора кластера (для `rac.exe`); RAS endpoint (`host:port`, по умолчанию `localhost:1545`); платформа 1С (путь к `rac.exe` и версия по умолчанию — выбираются единым пикером «Платформа 1С» из найденных установок или задаются по отдельности). |
| SQL Server | SQL-инстанс, на котором живут базы клиентов; используется в discovery при создании инфобазы. |
| Публикации IIS | Корневая папка публикаций (`IIS.DefaultVrdRoot`); сайт IIS по умолчанию (`IIS.DefaultSiteName`). |
| Опрос 1С | Частота полного опроса (`Polling.ColdIntervalSeconds`); частота опроса у лимита (`Polling.HotIntervalSeconds`); порог перехода на частый опрос (`Polling.HotThresholdPercent`); отсрочка enforcement (`Enforcement.KillGraceSeconds`); частота обновления статусов публикаций (`Drift.IntervalMinutes`). |
| Хранение данных | Срок хранения журнала аудита и истории лицензий (в днях). |
| Резервные копии | Папка для бэкапов, срок хранения файлов, максимум параллельных бэкапов, запас свободного места на диске. |

Каждый параметр редактируется по кнопке **Изменить** рядом со своей строкой; секретные значения (пароль кластера) хранятся зашифрованными и в открытом виде не отображаются (поле показывает «Задано (скрыто)»).

## 6. Бэкапы

### Запуск резервной копии

1. Перейдите в раздел **Базы**, найдите нужную инфобазу.
2. В строке нажмите **Бэкапы базы** (значок или действие в меню строки).
3. В открывшемся диалоге нажмите **Сделать бэкап**.
4. Система ставит задачу в очередь (статус «В очереди»), затем выполняет (`BACKUP DATABASE ... COPY_ONLY`). Список в диалоге обновляется каждые 5 секунд.

Кнопка **Сделать бэкап** доступна обеим ролям (Admin и Viewer).

Перед запуском система проверяет свободное место на диске SQL-сервера (оценка размера базы + запас `Backup.DiskSafetyMarginMb`). Папка для бэкапов должна быть задана в **Параметрах** → секция «Резервные копии», иначе появится ошибка «Папка бэкапов не настроена».

### Где смотреть статус и результат

В диалоге «Бэкапы базы» таблица содержит столбцы:

| Столбец | Описание |
|---|---|
| Статус | «В очереди», «Выполняется» (с анимацией), «Готов», «Ошибка» |
| Запрошен | Дата и время запроса |
| Завершён | Дата и время завершения |
| Размер | Размер файла бэкапа |
| Кем | Логин запустившего пользователя |

При статусе «Ошибка» под статусом отображается причина: нехватка места, нет прав SQL (`sysadmin`), прервано перезапуском сервиса и др.

Все события бэкапа фиксируются в разделе **Аудит**: «Бэкап запрошен», «Бэкап выполнен», «Бэкап не выполнен».

Хранится один актуальный файл на базу; устаревшие файлы удаляются автоматически ночью по параметру `Backup.TtlHours` (по умолчанию 24 часа).

### Кто может удалять бэкапы

Удалить бэкап из диалога может только **Администратор** (кнопка с иконкой корзины в строке). Кнопка заблокирована, пока бэкап выполняется (`Running`). Удаление удаляет файл с диска SQL-сервера и запись в панели — действие необратимо.

## 7. Операции IIS из панели

Управление IIS находится в разделе **Базы** → вкладка **IIS**.

### Пулы приложений

Для каждого пула показывается его текущее состояние: **Запущен**, **Остановлен**, **Запускается**, **Останавливается**, **Неизвестно**.

Доступные действия (только Admin):
- **Перезапустить пул** (recycle) — рабочий процесс пула перезапускается; текущие запросы к опубликованным в пуле базам прерываются. Требует подтверждения.
- **Запустить** — запуск остановленного пула; выполняется без подтверждения.
- **Остановить** — пул останавливается; базы в нём становятся недоступны до ручного запуска. Требует подтверждения.

### Сайты IIS

Для каждого сайта показывается состояние аналогично пулам.

Доступные действия (только Admin):
- **Запустить** — без подтверждения.
- **Остановить** — все публикации на сайте становятся недоступны. Требует подтверждения.
- **Перезапустить** — сайт кратковременно останавливается и запускается снова. Требует подтверждения.

### Глобальные операции IIS (только Admin)

Кнопки в заголовке карточки:
- **Перезапустить IIS** — выполняет `iisreset`; все сайты сервера кратковременно недоступны, включая саму панель, если она размещена на том же IIS. Требует подтверждения.
- **Остановить IIS** — выполняет `iisreset /stop`; все сайты полностью останавливаются. Кнопка **Запустить IIS** появляется вместо неё и запускает IIS обратно без подтверждения.

Все операции IIS фиксируются в журнале **Аудит** (типы «Пул приложений перезапущен», «Пул остановлен», «Сайт IIS остановлен», «Полный перезапуск IIS» и т. д.).

> **Важно.** Операции `Остановить IIS` и `Перезапустить IIS` прерывают работу всех клиентов хостинга. Выполняйте их в согласованные окна обслуживания.
