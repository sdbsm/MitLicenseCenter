## Актуальные планы (вне релиза 0.4.0-beta)

### 1. ESLint-правило для StatusBadge

`StatusBadge` — единственный способ отображать статусы в UI; правило его применения
поддерживается ревью, не линтером. Запланировано: кастомное ESLint-правило,
форсящее `StatusBadge` вместо сырых цветовых классов.

### 2. RAS Strategy B (оптимизация)

Текущее состояние: `rac.exe` запускается как отдельный процесс на каждый цикл.
Кросс-вызовой кэш UUID кластера уже сократил steady-state spawn rate примерно
вдвое; бюджет ≤ 26 proc/min выдерживается с запасом.

Следующий шаг — переход на долгоживущий TCP-сокет на порту 1545 (Strategy B).
Открывается после замера реальной латентности на production-нагрузке.

### 3. Multi-cluster / multi-node топология

Все адаптеры предполагают single-node (зафиксировано ADR-28). Расширение требует
полного пересмотра каждого адаптера и операционной модели. Брать только при явном
запросе со стороны оператора и после отзыва ADR-28.

### 4. Серверный фильтр «Базы» по метке «не найдена в кластере»

Метка «Не найдена в кластере» уже присутствует на странице Баз. Серверный фильтр
(join снапшота RAS в списочный запрос) делает отфильтрованный список быстрым при
большом числе баз. Брать по запросу оператора.

## Отложенные follow-up (брать по триггеру)

Точечные продолжения закрытого пострелизного трека — не активны, берутся по появлению триггера:

- **Low-priv аккаунт службы** (R9 / SEC-05, ADR-44): узкий ACE на `inetsrv\config` вместо
  членства службы в Administrators. Триггер — runtime-трасса реальных чтений `ServerManager`
  на стенде (понять минимально достаточный набор прав).
- **Optimistic concurrency на Infobase/Publication** (R12 / BE-02): сейчас rowversion-токен
  только на `Tenant`. Триггер — потребность защитить от одновременной правки и эти сущности.

## Завершено — пострелизный рефакторинг R1–R12

Трек `MLC-118..139` (2026-06-13..14) **закрыт полностью**: весь остаточный техдолг
пред-релизного аудита (`audit/2026-06/`) отработан по 12 итерациям — критический

видимость лимитов, устойчивость джобов и службы, релизный конвейер, hardening инсталлятора,
ревизия канона, масштабируемость списков, полировка UX (доступность WCAG AA, терминология,
онбординг).

Детали по каждой итерации и находке — в git (коммиты `MLC-NNN:` и PR), индекс закрытых задач —
`PROJECT_BACKLOG_ARCHIVE.md`, первоисточник находок — `audit/2026-06/TECH-DEBT.md`.
Открытые продолжения вынесены выше в «Отложенные follow-up».

только явным отзывом ADR-15.
